# Story Squares

**Story Squares** — веб-приложение для быстрого создания квадратных фото-историй/открыток из ваших снимков. Загружайте \~36 фотографий, раскладывайте по шаблону с разными слотами, добавляйте подписи и QR-ссылки, экспортируйте готовые карточки (JPEG/PNG) и общий PDF — всё локально, прямо в браузере.

> ⚠️ Проект в активной разработке (MVP). Этот README описывает цель и ближайший план; функциональность будет добавляться поэтапно.

---

## Зачем это нужно

* **Быстро** собрать серию квадратных карточек из набора фото.
* **Единый стиль**: фиксированные пресеты сетки/масок (слоты разного формата).
* **Подписи и QR**: пояснения к фото и ссылки (альбом/сайт/видео).
* **Экспорт офлайн**: готовые картинки + PDF без сервера и без сливов данных.

---

## Основные возможности (план MVP)

* Загрузка \~36 фотографий (drag-and-drop + выбор файлов).
* Авторазмещение по выбранному квадратному шаблону (слоты разных форм).
* Ручная подгонка: зум, позиция кадра, `cover/contain`, вкл/выкл масок.
* Подписи к фото и опциональный **QR-код** (генерация внутри приложения).
* Экспорт: пакет **JPEG/PNG** (на устройство/в ZIP) и общий **PDF**.
* Сохранение/загрузка проекта (локально, в браузере).
* Работа офлайн (PWA) и приватность: все данные остаются у пользователя.

---

## Стек

* **React + Vite + TypeScript**
* **Tailwind v4** (OKLCH-переменные, light/dark)
* **shadcn/ui** (готовые UI-примитивы)
* Далее по плану (MVP):

  * Хранилище: Zustand + IndexedDB (Dexie)
  * Экспорт PDF: `pdf-lib`
  * Экспорт изображений: Canvas / `html-to-image` + `pica`
  * QR-коды: `qrcode`
  * Сохранение файлов: File System Access API (с фоллбеком на ZIP)

## Быстрый старт

Требования: **Node 18+**

```bash
# установка
npm i

# режим разработки
npm run dev

# продакшен-сборка (в /docs для GitHub Pages)
npm run build

# предпросмотр собранной версии
npm run preview
```

---

## Деплой на GitHub Pages

Проект собирается в `docs/` и публикуется с базовым путём репозитория.

1. В `vite.config.ts` проверьте `base`:

```ts
// если репозиторий называется story-squares
base: '/story-squares/'
// если деплой не на Pages — можно поставить '/'
```

2. Выполните:

```bash
npm run build
```

3. В настройках репозитория включите **Pages** → Source: **/docs**.

## Дизайн и темы

* Готовые **light/dark/system** темы на OKLCH-токенах.
* Хук `useAutoThemeClass()` автоматически применяет системную тему и реагирует на смену `prefers-color-scheme`.
* Компоненты UI добавляются командой:

  ```bash
  npx shadcn@latest add button
  ```

---

## Дорожная карта (MVP)

1. **Импорт/превью**: загрузка фото, быстрые превью, базовая сетка.
2. **Редактор карточек**: зум/позиция, подписи, QR-код.
3. **Экспорт**: пакет JPEG/PNG (FS API/ZIP), общий PDF (`pdf-lib`).
4. **Хранение проекта**: IndexedDB + экспорт/импорт проекта (ZIP).
5. **Шаблоны**: несколько пресетов (круг/прямоуг./скругления), авто-раскладка 36 фото.
6. **PWA**: офлайн-режим, кэширование ассетов.

---

## Приватность

Все операции (рендер, QR, экспорт) выполняются **локально в браузере**. Фотографии не отправляются на сервер, если вы сами не публикуете собранные файлы.
